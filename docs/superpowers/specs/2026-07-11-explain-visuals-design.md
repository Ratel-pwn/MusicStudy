# 观察讲解图示设计

## 问题

课程中的八个 `explain` 步骤只显示确认按钮。步骤配置已经描述要讲解的音乐概念，渲染器没有读取这些配置，学习者看不到题目要求观察的对象。

## 目标

每个观察步骤必须呈现一张与课程配置一致的音乐理论图示。图示在桌面、平板和手机上均完整可读，支持键盘与屏幕阅读器，并在减少动效模式下保持稳定。

## 视觉系统

图示延续现有暖纸色、深海军蓝、金黄和薄荷绿配色。所有图示位于一张宽幅教学画布中，通过线条、键位、音点和拍格表达关系；不使用装饰性卡片组，也不采用标签、标题、描述的重复三段式结构。

八种图示分别为：

1. 竖直音高轨迹：低音点与高音点沿纵轴排列，箭头强调上行。
2. 中央 C 定位：钢琴键盘突出两黑键左侧的 C4。
3. 白键音名：展示 C 到 B 的白键顺序与回到 C 的循环。
4. 四分脉搏：四枚等距拍格完整填满 4/4 小节。
5. 八分细分：四个均匀位置展示“一、和、二、和”。
6. C 大调半音：C 到 C 的白键序列突出 E–F 与 B–C 两处半音。
7. C 大三和弦：音阶第 1、3、5 级通过连接线汇聚成 C–E–G。
8. 八小节结构：1–4 小节建立材料，5–8 小节重复并在结尾变化。

## 架构

新增独立 `ExplainVisual` 组件，接收 `LessonStep` 并依据已有配置键选择图示。`ExplainStep` 只负责组合图示与确认按钮。未知配置仍显示一张可读的通用概念画布，避免再次出现空白区域。

图形使用 React、HTML、CSS 与内联 SVG 构建，不增加网络资源或运行时依赖。动效使用现有 CSS 转场，关键内容拥有文本化 `aria-label`。

## 验收

- 八个已编写的观察步骤都渲染带名称的图示。
- 第一关第二步可见“低音”“高音”和上行轨迹。
- 点击“我看清了”后仍按现有引导流程进入下一步。
- 第一关六步能够完成并获得课程结算结果。
- 单元测试、复制检查、构建、桌面/平板/手机端到端测试通过。
