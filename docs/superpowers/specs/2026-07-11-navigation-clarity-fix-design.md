# 拾音岛导航清晰度修复设计

## 背景

Chrome 实测确认了四类问题：普通页面的主导航项目会随路由增减；悬停项目与当前项目使用相同的黄色填充；路由切换保留旧滚动位置；手机底部导航可能覆盖临近页面底部的操作按钮。群岛页还会因为固定导航与 `100vh` 地图叠加产生额外纵向滚动。

## 目标

- 让用户在普通页面始终看到稳定、可预测的五个主目的地。
- 让当前状态、悬停状态和键盘焦点拥有明确差异。
- 让每次主导航切换从新页面顶部开始。
- 消除群岛页标题与固定导航的遮挡。
- 为手机底部导航预留可靠的内容安全空间。

## 导航结构

首页、群岛、练习、创作入口和能力页统一展示以下顺序：

1. 首页
2. 群岛
3. 练习
4. 创作
5. 能力

品牌标记保留为首页入口。桌面导航移除“从第一枚声音出发”“航线已展开”“选择下一段声音”等模式说明，腾出稳定空间并减少含义重叠。

课程页继续使用“退出课程 / 课程进行中”的专注导航。创作工作室继续使用返回群岛入口与自身运输控制条，避免主导航挤占编辑空间。

## 视觉与交互状态

- 当前页面：日光黄色实心背景、深海色文字，保持最强识别度。
- 鼠标悬停：浅色半透明背景、纸白文字和轻微上移，不出现黄色实心背景。
- 键盘焦点：清晰的双层焦点环，独立于悬停和当前状态。
- 点击区域：桌面端维持至少 44px 高度；手机端维持至少 44px × 44px。
- 动效：使用短促的位移与颜色过渡；遵循 `prefers-reduced-motion`。

## 布局行为

### 桌面与平板

固定导航保持居中悬浮。所有普通页面统一预留顶部导航高度。群岛页使用扣除顶部安全区后的可视高度，地图内部拖动探索，页面本身不产生地图导致的纵向滚动。

### 手机

主导航保持底部五项布局。页面内容底部增加导航高度、系统安全区与额外操作缓冲的总空间，使提交、继续等按钮可以完整滚动到导航上方。触屏环境不保留桌面悬停状态。

## 路由滚动

在应用壳层加入路由滚动复位：`pathname` 改变时将窗口滚动到顶部。该行为仅作用于主页面滚动，不重置群岛地图的内部拖动偏移，也不干预浏览器前进后退以外的本地数据。

## 测试与验收

自动化测试覆盖：

- 首页、群岛、练习和能力页均渲染五个主目的地。
- 当前路由只产生一个 `is-active` 主导航项。
- 路由变化触发一次滚动复位。
- 课程与工作室专注导航保持原行为。
- 群岛页的视口高度使用导航安全区计算值。
- 手机样式保留五个可点击项目并提供足够底部间距。

Chrome 验收覆盖桌面宽屏与 390 × 844 手机视口：当前项和悬停项可立即区分；切换页面后标题完整可见；群岛页无多余纵向滚动；页面底部操作可以滚动到导航上方。

## 范围边界

本次不重做首页内容、群岛课程节点、练习任务泳道或工作室运输控制。导航信息架构、状态表现、滚动复位与安全间距之外的视觉调整留到后续迭代。
