# 拾音岛：零基础音乐学习平台设计规格

日期：2026-07-11  
状态：已确认
首版定位：本地优先、完整可玩的产品原型

## 1. 产品摘要

拾音岛面向 16–35 岁、没有音乐基础且希望逐步掌握写曲与编曲的学习者。产品以“音符群岛”为主视觉和学习骨架，将听觉、乐理、练习、复习与创作连成持续闭环。

首版交付一条真实可玩的核心路径：

```text
进入群岛地图
→ 完成互动课程
→ 接受即时判题与解释
→ 进入错题变式和间隔复习
→ 把新知识放入 8 小节作品
→ 保存本地进度与作品
→ 解锁下一关
```

首版验证核心交互和工程架构。完整课程内容、古典与流行双主线、云端能力和专业编曲能力按路线图逐步扩展。

## 2. 已确认的产品决策

- 核心受众：16–35 岁青少年与成年人。
- 首版层级：完整可玩原型，数据保存在当前设备。
- 产品主视觉：音符群岛，整体气质接近现代音乐绘本与探索地图。
- 学习结构：共同基础完成后选择主线。
- 课程方向：通用乐理基础，同时覆盖流行创作与古典素养。
- 双主线关系：用户选择一条主线后仍可随时切换，最终汇流到作曲与编曲毕业项目。
- 核心教学顺序：先听、再看、跟着做、辨一辨、自己搭、放进作品。
- 激励原则：保留 XP、连胜、星级、徽章与解锁；答错不会消耗体力或阻止继续学习。
- 创作评价：客观规则给出检查结果，审美建议保持建议语气。
- 开发设计技能：实现阶段同时应用 `frontend-design` 与 `gpt-taste`。

## 3. 文案与视觉硬约束

### 3.1 禁止项

- 禁止使用先否定再转折的模板句式。
- 禁止大面积重复“标签 + 标题 + 描述”三段式版面。
- 禁止通用 SaaS 仪表盘式侧栏、指标卡和同构卡片矩阵。
- 禁止紫色渐变白底、过量玻璃拟态、无意义悬浮徽章等常见生成式界面套路。
- 禁止用大量文字解释本可通过声音、谱面、琴键、节奏格或时间轴表达的信息。
- 禁止将“好不好听”伪装为确定性的机器评分。

### 3.2 正向规则

- 地图承担课程导航。
- 五线谱、钢琴键盘、和弦垫、节奏格与时间轴承担主要信息表达。
- 不同页面拥有各自的空间逻辑和交互重心。
- 深海蓝作为主背景色；暖黄、珊瑚红、薄荷绿分别表达进度、动作与正确反馈。
- 标题使用具有书卷感的衬线字体，正文使用清晰的中文无衬线字体。
- 动效服从节拍、播放、解锁和反馈，避免装饰性漂浮。
- 大标题控制在两至三行内，保证宽阔布局和充足留白。
- 所有按钮保持明确的文本对比度、键盘焦点和触控尺寸。

## 4. 学习旅程

### 4.1 共同基础群岛

1. 音符坐标
   - 音高与声音高低
   - 键盘布局、音名、八度
   - 五线谱、谱号、升降号
   - 全音、半音与基础音程
2. 节奏工坊
   - 拍点与速度
   - 四分、八分、二分、全音符与对应休止
   - 小节、4/4 拍、细分与重音
   - 两小节节奏复刻
3. 调性色谱
   - 全音和半音排列
   - 大调音阶、自然小调
   - 主音、级数、关系大小调
   - 在不同主音上构建音阶
4. 和弦实验室
   - 三和弦构成
   - 大、小、减三和弦
   - 调内和弦与罗马数字
   - 基础转位与听觉差异

### 4.2 流行创作航线

- 常见四和弦进行与移调。
- 主、下属、属功能和终止感。
- 旋律动机、重复、变化与乐句问答。
- 鼓、贝斯、和弦、旋律的声部职责。
- Lo-fi、电子、流行与影视配乐的基础模板。
- 段落、动态、留白和基础编曲。

### 4.3 古典素养航线

- 读谱速度和谱面定位强化。
- 单音、音程、节奏与和弦的视唱练耳。
- 功能和声、终止式与声部进行。
- 动机发展、乐句结构与常见曲式。
- 二声部关系、音域和织体。
- 钢琴小品或室内乐缩编的基础写作。

### 4.4 汇流创作山

两条航线都汇入毕业项目。长期毕业要求为：

- 完成至少 16 小节作品。
- 包含鼓或节奏层、低音层、和声层和旋律层。
- 说明作品的调、和弦进行、动机与段落结构。
- 通过节拍、调内音、和声功能、音域、结构和终止检查。
- 根据系统建议完成至少一次修改。

## 5. 首版 MVP 范围

### 5.1 首版可玩内容

- 欢迎与快速听音体验：用户进入 30 秒内能点击第一个音并听到反馈。
- 群岛课程地图：显示完整长期路线，首版开放一条代表性学习链。
- 互动课程：至少 8 个可完整通关的节点，覆盖音高、键盘、基础节奏、大调音阶和三和弦。
- 互动题库：至少 6 类题型，每类提供难度递进和错误解释。
- 今日练习：组合到期复习、近期薄弱题和一个作品修改任务。
- 错题系统：记录技能标签、错误类型、提示次数和下一次复习时间。
- XP、星级、连续学习和世界徽章。
- 8 小节四轨创作台：鼓、贝斯、和弦与旋律。
- 本地作品保存、回放、继续编辑和重命名。
- 学习进度、设置与作品通过 IndexedDB 持久化。
- 桌面端完整体验与移动端可用体验。

### 5.2 首版内容节点建议

1. 听见高与低。
2. 找到中央 C。
3. 认识白键音名。
4. 四分音符与稳定拍点。
5. 八分音符与节拍细分。
6. 用全音和半音搭出 C 大调。
7. 用三个音搭出 C 大三和弦。
8. 毕业挑战：制作 8 小节循环。

地图同时显示后续世界轮廓，未开放节点展示明确的能力目标和迭代状态。

### 5.3 首版暂缓

- 账号登录与跨设备同步。
- 教师、家长或管理员后台。
- 真人课程视频和版权曲库。
- 麦克风唱音识别和真实乐器录音。
- MIDI 键盘输入。
- 社区、关注、排行榜、评论和公开作品。
- AI 对话教师。
- 完整五线谱排版引擎。
- DAW 工程兼容、插件、自动化与专业混音。
- WAV、MIDI、MusicXML 导出。

## 6. 核心页面与独特空间结构

### 6.1 开场与首页

- 首次进入直接展示可点击琴键和可听见的短旋律。
- 用户完成一次操作后再出现产品解释和开始按钮。
- 返回用户直接进入当前群岛位置。
- 首页保留宽阔的地图视野，导航融入海图边缘和航线标记。

### 6.2 群岛地图

- 课程节点沿航线分布，岛屿形态表达不同知识类型。
- 节点状态包含未解锁、可学习、进行中、已完成、需要复习和作品关。
- 点击节点先试听课程声音样本，再展开开始入口。
- 地图允许拖动和缩放；键盘用户可按课程顺序移动焦点。

### 6.3 统一课程播放器

- 顶部只保留退出、当前进度和音频状态。
- 中央区域根据步骤切换谱面、琴键、节奏格或和弦构建器。
- 底部动作区承担播放、提交、提示和继续。
- 每节固定六步，组件由课程 JSON 配置驱动。
- 每个概念至少包含一个听觉动作和一个构造动作。

### 6.4 练习中心

- 使用任务泳道表达练习顺序和完成状态。
- 今日练习由 3 道到期复习、2 道薄弱题和 1 个作品任务组成。
- 提供 5 分钟快练与按技能选择的 10 分钟专项。
- 练习结束用能力轨道展示变化，并给出下一次复习日期。

### 6.5 四轨创作台

- 时间轴占据核心区域，减少工具栏和设置面板占比。
- 鼓轨使用步进音序器。
- 贝斯轨与旋律轨使用简化钢琴卷帘。
- 和弦轨使用和弦垫和拖拽片段。
- 每次修改可以立即播放当前循环。
- 规则检查以可操作建议落在具体小节和音符上。
- 首版固定 4/4 拍，可选 70、90、110、128 BPM，支持 C 大调和 A 自然小调。

### 6.6 能力与复习页

- 使用环形技能轨道表达音符、节奏、听力、音阶、和弦和创作能力。
- 展示近期进步、待复习技能和作品里程碑。
- 不展示与学习决策无关的虚荣指标。

## 7. 题型与反馈

### 7.1 首版题型

1. 键盘定位：点击指定音名或听到的音。
2. 高低判断：比较两个音的高低关系。
3. 节奏跟拍：在目标拍点触发输入。
4. 节奏排序：将时值拖入小节并满足拍数。
5. 音阶构造：在键盘中选择组成音。
6. 和弦构造：选择根音、三音和五音。
7. 错误修复：替换旋律或和弦中的异常元素。
8. 受限创作：满足调、拍数、音域和轨道要求。

### 7.2 判题反馈层级

1. 第一次错误：指出关注方向，不显示答案。
2. 第二次错误：高亮相关琴键、谱线、拍点或音程距离。
3. 第三次错误：演示正确过程，随后生成一道同构变式题。

反馈示例：

> C 到 E♭ 包含 3 个半音，因此形成小三度。大三度需要 4 个半音，请再向上移动一个半音。

### 7.3 间隔复习

- 初始复习间隔：1、3、7、14 天。
- 连续正确且未使用提示时延长间隔。
- 使用提示或同类错误重复出现时缩短间隔。
- 首版采用可解释规则，不使用黑盒推荐模型。

## 8. 激励与通关

- 1 星：完成课程。
- 2 星：正确率达到 80%，提示使用较少。
- 3 星：完成迁移题或微创作任务。
- 世界解锁：所有必修节点至少 1 星，作品关达到 70 分。
- 未通过作品关时生成 2–4 道针对性补练，完成后可立即重试。
- XP 表示有效练习量。
- 连续学习要求完成一课或至少五道复习题。
- 章节奖励为乐器、音色、地图区域和创作台主题。

## 9. 工程架构

### 9.1 技术栈

- React + TypeScript + Vite。
- React Router：地图、课程、练习、创作台和能力页路由。
- Zustand：会话状态、课程运行状态、音频状态和轻量 UI 状态。
- Dexie + IndexedDB：进度、复习记录、设置和作品持久化。
- Tone.js：合成器、采样播放、节拍器、Transport 和量化调度。
- GSAP + `@gsap/react`：地图进入、路径解锁、节拍反馈与关键过渡。
- Tailwind CSS 与设计令牌：布局、状态和响应式样式；复杂音乐组件使用组件级 CSS。
- Phosphor Icons：通用控制图标。
- Vitest + Testing Library：单元与组件测试。
- Playwright：关键用户旅程端到端测试。

### 9.2 模块边界

```text
src/app            路由、布局、启动与错误边界
src/features/map   学习地图和节点解锁
src/features/lesson 课程播放器、步骤状态机和判题
src/features/practice 复习调度、专项练习和能力更新
src/features/studio 四轨创作台、规则检查和作品保存
src/features/profile XP、连续学习、徽章和能力画像
src/audio          Tone.js 封装、音色、调度和音频上下文
src/music          音高、节奏、音阶、和弦和量化等纯函数
src/content        课程、题目、技能标签与地图配置
src/storage        Dexie 表、迁移和仓储接口
src/design         设计令牌、动效参数和通用原子组件
```

每个 feature 只通过公开接口访问音乐规则、音频和存储。课程内容不直接操作音频实例，UI 不直接写 IndexedDB。

### 9.3 关键数据流

```text
课程 JSON
→ 课程步骤状态机
→ 交互组件提交标准化答案
→ music 规则层判定
→ 反馈与音频层同步呈现
→ 记录 attempt 和 skill mastery
→ practice 调度下次复习
→ map 根据 mastery 和星级更新解锁
```

### 9.4 核心数据实体

- `CourseWorld`：世界、顺序、主题、前置条件。
- `Lesson`：标题、技能标签、步骤、奖励、解锁规则。
- `LessonStep`：类型、提示、音频配置、交互配置和判题配置。
- `Attempt`：答案、正确性、提示层级、耗时和错误类型。
- `SkillProgress`：熟练度、连对次数、下次复习时间和最近错误。
- `UserProgress`：XP、连续学习、星级、徽章和当前位置。
- `Composition`：速度、调性、小节、轨道、片段和更新时间。
- `Settings`：音量、减少动效、记谱偏好和输入偏好。

## 10. 音频设计

- 浏览器首次用户操作后创建并恢复 AudioContext。
- 所有节拍与音符播放通过 Tone.Transport 调度，避免依赖普通定时器。
- 首版提供钢琴、柔和合成器、贝斯、鼓组和节拍器。
- 首次加载优先使用轻量合成音色；采样音色按需加载。
- 交互音与教学音分配独立音量通道。
- 页面隐藏或设备休眠后暂停播放，恢复时要求用户主动继续。
- 音频不可用时保留视觉练习，并给出明确的恢复入口。

## 11. 本地数据与未来云迁移

- IndexedDB 使用版本化 schema。
- 存储访问通过 repository 接口隔离。
- 数据实体预留稳定 UUID、创建时间、更新时间与同步状态。
- 首版提供导出和导入本地备份的结构基础，界面可延后。
- 云端阶段用同一领域模型映射服务端 API，减少功能层改动。
- 每次 schema 升级包含迁移测试和失败回滚策略。

## 12. 错误处理与恢复

- 应用级错误边界：保留导航和本地数据恢复入口。
- 课程内容错误：跳过损坏步骤并记录诊断信息，避免整课卡死。
- 音频初始化失败：显示一键重试，支持静音继续视觉题。
- 音色加载失败：回退到基础合成器。
- IndexedDB 写入失败：保留内存快照并提示导出临时备份。
- 作品数据损坏：打开最近一次自动保存快照。
- 拖拽失败或触控取消：元素回到原位置，状态保持一致。
- 页面刷新：恢复最近课程步骤或作品编辑位置。
- 所有错误文案说明影响、当前保存状态和下一步动作。

## 13. 无障碍与输入方式

- 支持键盘完成课程、练习和创作台核心操作。
- 琴键、音符、节奏块和和弦块提供可读标签。
- 颜色之外同时使用形状、位置和文字表达状态。
- 提供减少动效开关；系统开启 `prefers-reduced-motion` 时默认生效。
- 音频题支持重复播放、音量调节和视觉辅助模式。
- 触控目标最小 44×44 CSS 像素。
- 移动端创作台采用分轨聚焦编辑，避免缩小整个桌面时间轴。

## 14. 测试策略

### 14.1 音乐规则单元测试

- 音名与 MIDI 映射。
- 全音、半音和音程计算。
- 大调、自然小调生成。
- 大、小、减三和弦构建。
- 小节拍数和节奏合法性。
- 量化、移调和音域检查。
- 作品规则检查。

### 14.2 课程引擎测试

- 六步课程状态机。
- 提示层级递进。
- 变式题生成与完成条件。
- 星级、XP 和解锁规则。
- 间隔复习调度。

### 14.3 组件测试

- 钢琴键盘、节奏格、和弦构建器和钢琴卷帘。
- 鼠标、触控和键盘输入一致性。
- 音频关闭时的视觉降级。
- IndexedDB 读写和 schema 迁移。

### 14.4 端到端测试

1. 新用户进入后播放第一个音。
2. 完成一个课程节点并获得星级。
3. 答错、使用提示、完成变式题。
4. 今日练习更新技能和复习日期。
5. 创建、保存、刷新并恢复 8 小节作品。
6. 完成作品挑战并解锁下一节点。
7. 开启减少动效并通过键盘完成关键路径。

### 14.5 设计质量检查

- 扫描禁用句式和未完成文案。
- 检查重复三段式结构和同构卡片滥用。
- 验证标题行数、按钮对比度、焦点状态和移动端溢出。
- 验证 GSAP 动效清理、减少动效路径与页面离开后的音频停止。

## 15. 首版验收标准

- 新用户 30 秒内听到并操作第一个音。
- 新用户 15 分钟内完成第一个可循环播放的四小节片段。
- 至少 8 个课程节点能够完整通关。
- 每个理论节点至少包含一道听觉题和一道构造题。
- 六类以上题型具备真实判定、解释、重试和变式流程。
- 课程、练习、进度和作品刷新后可恢复。
- 8 小节四轨作品可编辑、保存、回放和继续修改。
- 音频失败时视觉练习仍可继续。
- 桌面和移动端核心路径可用。
- 键盘用户可以完成主要学习路径。
- 自动化测试覆盖音乐规则、课程引擎和关键用户旅程。
- 生产构建通过，控制台无阻断性错误。

## 16. 迭代路线图

### 阶段 0：产品原型与技术闭环

目标：验证“学习—练习—复习—创作—解锁”闭环。

交付：

- 本规格第 5 节定义的 MVP。
- 代表性基础课程和 8 小节四轨创作台。
- 本地进度、复习和作品持久化。
- 音符群岛设计系统和核心动效语言。

退出条件：首版验收标准全部满足。

### 阶段 1：共同基础课程完整版

目标：让用户系统掌握进入双主线所需的共同语言。

交付：

- 四个共同基础世界，每个世界约 4 节核心课和 1 个作品关。
- 20–28 节完整课程。
- 听觉题、构造题、节奏题和微创作题库扩充。
- 内容校验器、课程预览工具和内部题目编辑流程。
- 更完整的能力画像和复习解释。

依赖：阶段 0 的课程引擎、内容 schema 和音频层稳定。

退出条件：用户完成基础线后能够读懂常见音名和节奏，构建大调、自然小调和调内三和弦。

### 阶段 2：流行创作与古典素养双主线

目标：支持不同学习目标并保持知识互通。

交付：

- 流行线：和弦进行、旋律、Beat、贝斯、段落与基础编曲。
- 古典线：读谱、视唱练耳、功能和声、乐句、曲式和二声部。
- 主线切换、跨线推荐与共同技能映射。
- 16 小节毕业项目和结构化作品自述。
- 流行、Lo-fi、电子、影视和古典小品声音模板。

依赖：阶段 1 的基础课程完成度和内容生产工具。

退出条件：用户能独立完成并解释一首 16 小节作品。

### 阶段 3：创作与编曲工作室增强

目标：从教学工具成长为可持续使用的轻量音乐工作室。

交付：

- 8–16 轨、段落编排、循环库、速度与调性扩展。
- 七和弦、借用和弦、次属和弦、调式和转调工具。
- 自动和声建议、音域冲突提示、动机追踪和版本历史。
- MIDI 输入与 MIDI 导出。
- WAV 导出和基础混音控制。
- 桌面键盘快捷键和触屏创作模式。

依赖：稳定的 Composition 领域模型和音频调度。

退出条件：用户可以在平台内完成结构清晰、可导出的完整 Demo。

### 阶段 4：云端正式产品

目标：提供跨设备、可运营和可持续扩展的正式服务。

交付：

- 账号、登录、跨设备同步和冲突处理。
- 课程、题目、音色和地图配置后台。
- 作品云端保存、版本和私密分享。
- 学习数据分析、课程漏斗和内容质量监控。
- 订阅、权益、订单和合规基础。
- 隐私、数据导出、账号删除和未成年人保护策略。

依赖：本地 repository 接口、稳定数据模型和内容校验流程。

退出条件：可安全运营、发布课程并支持真实用户规模。

### 阶段 5：智能辅导与音乐生态

目标：提高个性化学习效率并连接真实创作场景。

交付：

- AI 辅导员读取课程上下文、技能记录和作品规则结果。
- 可解释的个性化复习和课程推荐。
- 麦克风唱音、节奏与真实乐器练习。
- 教师反馈、班级、作业与学习报告。
- 作品社区、挑战、合作与授权安全机制。
- 原生移动端、离线课程和桌面应用评估。

依赖：云端身份、权限、内容安全、音频隐私和成本控制。

退出条件：智能功能在准确性、可解释性、隐私与成本方面达到生产标准。

## 17. 实施顺序

1. 初始化 React、TypeScript、测试与设计系统。
2. 实现纯函数音乐规则层并先完成测试。
3. 实现 AudioContext、Transport、乐器和节拍器封装。
4. 实现 IndexedDB schema、repository 接口和迁移测试。
5. 实现课程内容 schema、步骤状态机和统一反馈模型。
6. 实现钢琴键盘、节奏格、和弦构建器与基础谱面。
7. 实现群岛地图、课程解锁、XP、星级和连续学习。
8. 实现练习调度、错题变式和能力画像。
9. 实现 8 小节四轨创作台和作品规则检查。
10. 写入首版课程内容和毕业挑战。
11. 完成响应式、键盘操作、减少动效和错误恢复。
12. 运行单元、组件、端到端、构建与设计质量检查。

## 18. 成功判断

首版成功以用户行为和能力证明为依据：

- 用户愿意完成第一条学习链。
- 用户能解释一次错误的原因并在变式题中改正。
- 用户能在 15 分钟内产出可回放的短循环。
- 用户理解课程知识与作品之间的连接。
- 用户刷新页面后愿意继续当前进度。
- 产品视觉具有清晰识别度，界面不依赖常见生成式模板。

该设计为首版提供严格边界，同时为课程扩充、双主线、专业创作、云端服务与智能辅导保留稳定的演进路径。
